# 部门配置领域知识

这个 skill 只提供领域知识，不负责执行链路。

## 访问边界

- `DeptManager`：只处理自己部门
- `Admin`：可跨部门
- `CSR` 和 `TeamLeader`：不进入本域

## 常见意图

### 1. 查看部门配置

- 意图关键词：部门配置、部门信息、我的部门设置、部门详情
- API 查找种子：`部门详情`、`departments {id}`
- 默认范围：当前用户部门
- 期望输出：负责人、邮箱、AI 绑定、描述、基础配置

### 2. 更新负责人或基础信息

- 意图关键词：换负责人、改部门经理、修改部门名称、修改描述、修改编码
- API 查找种子：`更新部门`、`部门负责人`、`manager`
- 常见辅助查询：员工搜索、部门列表、HR org manager 关系

### 3. 邮箱配置

- 意图关键词：换邮箱、修改部门邮箱、绑定新邮箱
- API 查找种子：`emails page`、`创建邮箱配置`、`更新邮箱配置`
- 关键输入：目标部门、新邮箱
- 风险：可能覆盖已有绑定，主 agent 需要确认

### 4. Atlas / AI 绑定

- 意图关键词：开启 Atlas、关闭 Atlas、AI 绑定、Atlas AI
- API 查找种子：`ai agents page`、`更新部门 aiAgentId`
- 常见辅助查询：可用 AI agent 列表

### 5. 部门列表与排班

- 意图关键词：有哪些部门、部门列表、排班、工作时间
- API 查找种子：`departments page`、`schedule`
- 用途：做跨部门选择或查看开放时间

## 额外约束

- 非 Admin 遇到跨部门配置请求时直接拦截
- 涉及负责人、邮箱、AI 绑定时，优先先查当前状态再改
- 变更类请求默认视为写操作，需要确认
