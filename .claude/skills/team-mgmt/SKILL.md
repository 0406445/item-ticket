# 团队管理

提供团队成员管理相关的业务知识和 API 查找线索。
适用于成员增删、员工入组和员工信息类请求，不负责编排或执行。

## 访问边界

- `TeamLeader` / `DeptManager`：只处理自己部门
- `Admin`：可跨部门
- `CSR`：不进入本域

## 常见意图

### 1. 添加或移除成员

- 意图关键词：添加成员、加入团队、移除成员、从团队删除
- API 查找种子：`departments members`、`添加成员`、`移除成员`
- 关键输入：目标员工、目标部门
- 常见辅助查询：员工搜索、部门搜索
- 风险：移除成员是不可逆写操作

### 2. 创建员工并入组

- 意图关键词：新人入职、创建新员工并加入、把新员工加到团队
- 典型拆分：先创建员工，再添加到部门
- API 查找种子：`创建员工`、`departments members`
- 关键输入：姓名、邮箱、目标部门

### 3. 查询团队与员工信息

- 意图关键词：团队成员、成员列表、员工详情、查一下某人信息
- API 查找种子：`staff page`、`staff detail`、`options batch`
- 常见输出：姓名、邮箱、部门、状态

### 4. 更新员工信息

- 意图关键词：修改员工邮箱、更新员工信息
- API 查找种子：`更新员工`、`staff {id}`
- 关键输入：目标员工、修改内容

## 名称解析偏好

- 人员名称优先走员工分页查询
- 部门名称优先走部门分页查询
- “我的团队”“我的部门”默认落到当前用户部门

## 复合操作提示

- “把张三从技术部移到运营部”应拆为：解析人 -> 从原部门移除 -> 加入新部门
- “先建人再入组”要按顺序执行，前一步返回的员工 ID 传给后一步
- 对删除/移除类动作，主 agent 必须确认
