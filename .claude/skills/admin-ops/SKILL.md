# 管理员操作

当需要执行系统级管理操作时，参考本领域知识。这些操作仅 Admin 角色可用。

执行流程：通过 @findapiagent 查找 API → @api-validator 校验参数 → /api-executor 执行请求。

## 权限要求

本域所有操作仅限 **Admin** 角色。其他角色请求时，回复："这个操作需要管理员权限，你目前是 [当前角色]。如果需要，可以联系你的管理员。"

## 操作清单

### 1. 创建部门

- **用户意图关键词**：创建部门、新建部门、添加部门
- **需要的信息**：部门名称（必填）
- **API**：POST /v1/staff/departments [staff]
- **可选信息**：部门描述、部门编码、上级部门、关联 SLA、部门经理
- **说明**：如果用户只提供了名称，先用名称创建，其他信息后续可以补充。不要一次问太多信息。
- **执行**：通过 @findapiagent 查找创建部门 API，填充 name 参数后执行
- **后续建议**：创建成功后建议"要给这个部门添加成员吗？"

### 2. 创建员工账号

- **用户意图关键词**：创建员工、新建用户、添加账号、注册员工
- **需要的信息**：姓名（必填）+ 邮箱（必填）
- **API**：POST /v1/staff/staff [staff]
- **缺少信息时**：
  - 没有姓名 → "请提供员工姓名"
  - 没有邮箱 → "请提供员工邮箱"
  - 每次只问一个缺失信息
- **后续建议**：创建成功后建议"要把这个员工加到某个部门吗？"

### 3. 重置员工密码

- **用户意图关键词**：重置密码、初始化密码、密码重置
- **需要的信息**：员工（姓名或 ID）
- **API**：PUT /v1/staff/staff/{id}/password/reset [staff]
- **⚠️ 不可逆操作**：执行前必须请求用户确认
  - 确认格式："即将重置 [员工姓名] 的密码，确认吗？"
- **名称解析**：如果用户提供姓名，先通过 POST /v1/staff/staff/page [staff] 搜索解析为员工 ID

### 4. 创建 Topic（工单主题）

- **用户意图关键词**：创建主题、新建 Topic、添加工单分类
- **需要的信息**：Topic 标题（必填）+ 归属部门（必填）
- **API**：POST /v1/staff/topics [staff]
- **名称解析**：如果用户提供部门名称，先解析为部门 ID
- **缺少信息时**：
  - 没有标题 → "请提供主题标题"
  - 没有部门 → "这个主题归属哪个部门？"

### 5. 更新 Topic

- **用户意图关键词**：修改主题、更新 Topic、改主题名称
- **需要的信息**：Topic（名称或 ID）+ 要修改的内容
- **API**：PUT /v1/staff/topics/{id} [staff]
- **名称解析**：如果用户提供 Topic 名称，先通过 POST /v1/staff/topics/page [staff] 搜索解析为 Topic ID

### 6. 查询 Topic 列表

- **用户意图关键词**：有哪些主题、Topic 列表、工单分类列表
- **API**：POST /v1/staff/topics/page [staff]
- **结果展示**：
  ```
  当前系统有以下工单主题：
  1. 物流查询 — 归属：运营部
  2. 退款申请 — 归属：财务部
  3. 技术支持 — 归属：技术部
  ...
  ```

### 7. 系统概览

- **用户意图关键词**：系统概览、系统状态、有多少部门、有多少用户
- **部门总数**：POST /v1/staff/departments/page [staff]（获取 total）
- **用户总数**：POST /v1/staff/staff/page [staff]（获取 total）
- **AI 仪表盘**：POST /v1/staff/ai-dashboard/overview [staff]
- **结果展示**：
  ```
  系统概览：
  - 部门总数：X 个
  - 员工总数：X 人
  - AI 相关统计：...
  ```

### 8. 系统配置查询/更新

- **用户意图关键词**：系统配置、查看配置、修改配置
- **查询配置**：GET /v1/staff/configs/{namespace} [staff]
- **更新配置**：PUT /v1/staff/configs/{namespace}/batch [staff]
- **说明**：配置按 namespace 分组，需要用户指定要查看或修改的配置类别
